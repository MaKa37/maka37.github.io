# Making a GitHub Pages Blog Searchable on Google, Naver
- 깃허브 페이지를 구글과 네이버에 검색되게 하는 방법에 대한 포스트 입니다.
- 포스트 내용은 `Minimal Mistakes` 테마를 사용하는 것을 전제로 하고있습니다.
- 블로그 내 검색 기능 추가 방법도 포함합니다.

***

## Google
- 구글 검색 엔진 소유권 인증 및 사이트맵 제출 방법 정리
- 등록 완료되면 검색 시 블로그가 노출되게 됩니다.

### Ch 01. Google Search Console에 사이트 등록
1. Google Search Console에 접속하여 로그인
2. URL 접두어' 선택 상자에 블로그 주소(https://username.github.io)를 입력하고 [계속]을 누릅니다.
3. '소유권 확인' 창이 뜨면 여러 방법 중 **'HTML 태그'**를 선택합니다.
4. `<meta name="google-site-verification" content="알파벳숫자조합" />` 형태의 코드가 나옵니다.
5. 여기서 **content 뒷부분의 값(알파벳+숫자 조합)**만 복사해 두세요.

### Ch 02. 블로그에 인증 코드 넣기
1. Minimal Mistakes 테마는 설정을 통해 인증 코드를 쉽게 넣을 수 있도록 설계되어 있습니다.
2. 내 레포지토리의 _config.yml 파일을 엽니다.
3. 파일 내에서 google_site_verification 항목을 찾거나 없으면 추가합니다.
4. 아까 복사한 값을 따옴표 안에 넣습니다.
5. 수정 후 `Git Push` (블로그의 `<head>` 태그에 인증 코드가 자동 삽입됨.) 
6. 다시 구글 서치 콘솔 화면으로 돌아가 [확인] 버튼을 누릅니다. (소유권 인증)

```yaml
# 소유권 인증 번호 블로그 코드에 넣기.
# SEO Related
google_site_verification : # content(Alp + Num)
bing_site_verification   : # content(Alp + Num)
naver_site_verification  : # content(Alp + Num)
yandex_site_verification : # content(Alp + Num)
baidu_site_verification  : # content(Alp + Num)

# username.github.io 사이트 인증 번호 등록 확인 방법
# 블로그 접속 후 `Ctrl + U` 입력 후 `Ctrl + F` 후 google-site-verification에 content 내용을 확인한다.
```

### Ch 03. 사이트맵 등록 방법
1. 깃허브 블로그 테마인 `Minimal Mistakes`는 기본적으로 `jekyll-sitemap` 플러그인을 지원합니다.
2. 구글 서치 콘솔 왼쪽 메뉴에서 **[Sitemaps]**를 클릭합니다.
3. '새 사이트맵 추가' 입력창에 sitemap.xml이라고 입력하고 [제출]을 누릅니다.
4. 상태가 **'성공'**으로 뜨면 끝입니다. (처음에는 '가져올 수 없음'이 뜰 수 있는데, 몇 분 후 새로고침하면 성공으로 바뀝니다.)

## Naver
- 네이버 검색 엔진 소유권 인증 및 사이트맵 제출 방법
- 등록 완료되면 검색 시 블로그가 노출되게 됩니다.

***

## Search function within blog
- Github Blog 내 검색 기능 추가 방법

```yaml
#`maka37.github.io\_config.yml`이 경로에서 아래의 코드를 찾아 값을 변경한다.
search                   : true # true, false (default)
search_full_content      : true # true, false (default)
search_provider          : lunr # lunr (default), algolia, google
```
