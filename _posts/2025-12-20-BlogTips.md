# Github Blog Layout Tips

## Menu
- 메뉴 관리 방법은 아래와 같다.

```yaml
# `username.github.io\_data\navigation.yml` 경로에서 메뉴를 수정할 수 있다.
Menu
 ├─ Posts        (Page)
 ├─ Series       (Page)
 ├─ Categories   (Page)
 └─ About        (Page)

Post
 ├─ categories
 └─ series

# 메뉴 관리 코드 예시
main:
  - title: "HOME"
    url: /
  - title: "Posts"
    url: /posts/
```

## Post
- Structured of Post: 
  - 게시글을 의미하며 날짜 기반 정렬
  - RSS/최근 글
  - 카테고리·태그·시리즈는 front matter 데이터로 관리한다.
- 포스트 업로드 및 관리 방법
  - `_posts` 폴더에 게시글을 업로드하고 front matter 데이터로 관리하는 것이 핵심.
  - `_posts` 폴더가 없을 시 경로(`username.github.io/_posts`)에 폴더를 생성한다.
  - `_posts` 게시글을 업로드 하는 방법은 아래의 코드 블록을 참고한다.

```yaml
# post 폴더 내부에 아래의 마크다운 형식 파일을 생성하여 깃허브에 업로드한다.
# File Name: YYYY-MM-DD-title.md
_posts/
 ├─ 2025-01-10-git-basic-1.md
 ├─ 2025-01-12-git-basic-2.md
 └─ 2025-02-01-git-basic-3.md
```

### Front Matter Datas
- Front Matter란? Markdown 파일 맨 위에 위치하는 YAML 형식의 "메타데이터 블록"이다.
- 아래의 메타데이터를 통해 정보를 표기할 수 있다.
- 마크다운 파일의 구조
  1. Front Matter Block
  2. Contents

```yaml
# Front Matter 메타데이터 블록
# Post의 제목, 설명
title: "Post Title"
description: "Post description"

# 분류, 정렬, 구조 형성
date: 2025-12-31 # 보통 파일명 날짜를 기반으로 자동 정렬되어 생략함
categories: [BlogTips]
categories: 
  - BlogTips
  - Jekyll
series: BlogTips
order: 1 # 시리즈의 내부 순서 / 날짜보다 우선하는 명시적 정렬 기준

# 검색용 세부 키워드
tags: [github, blog, jekyll]

# 레이아웃 & 표현
laygout: single
classes: wide
author_profile: true # 좌측 프로필 표시 여부

# UI
toc: true # Table of Contents 활성화 여부
toc_label: "Toc Title"
toc_icon: "heart" # https://fontawesome.com/icons
toc_sticky: true # 스크롤 고정 여부
toc_levels: 2..3 # 최대 Header 표시 범위(h2~h3)

# Page Options
permalink: /blog-guide/ # URL 고정, 포트폴리오 페이지
sidebar:
  nav: "main" # 화면 좌측 메뉴 연결

# 동작 제어
published: false # 빌드 제외, 임시 글
comments: false # 댓글 비활성화
search: false # 검색 결과 제외
```
